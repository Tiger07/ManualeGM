Preferenze

Nel menu File -> Preferences puoi impostare un certo numero di impostazioni che verranno mantenute tra le varie esecuzioni di GameMaker. Possono essere impostate le seguenti preferenze:

Linguetta General

  • Show recently edited games in the file menu. Se selezionato gli otto giochi modificati più recentemente vengono mostrati nel menu file.
  • Load last opened file on startup. Se selezionato, quando aprirai GameMaker il file aperto più recentemente verrà aperto automaticamente.
  • Keep backup copies of files. Se selezionato, il programma salverà una copia di backup del tuo gioco con estensione gb0-gb9. Puoi aprire questi files su GameMaker. Dovresti usare almeno una copia backup nei tuoi giochi.
  • Maximum number of backups. Qui puoi indicare quante copie di backup (1-9) devono essere salvate dal programma.
  • Show progress while loading and saving files. Se selezionato, quando un gioco verrà salvato o caricato, verrà mostrata una barra che indica il progresso del caricamento.
  • At startup check for, and remove old temporary files. GameMaker e i giochi creati con esso, creano dei file temporanei. Normalmente questi vengono rimossi automaticamente, ma, per esempio quando un gioco crasha, vengono lasciati. Se questa opzione viene selezionata, GameMaker controllerà se ci sono questi file e li rimuoverà.
  • Don't show the website in the main window. Se selezionata, l'immagine e il link al sito web non verranno mostrati.
  • Hide the designer and wait while the game is running. Se selezionata, il programma verrà nascosto mentre stai testando un gioco.
  • Run games in secure mode. Se selezionata, ogni gioco creato con GameMaker che viene eseguito nel tuo computer verrà impedito dall'eseguire programmi esterni o cambiare e cancellare files in un posto differente dalla cartella del gioco. (Questo può essere usato contro i Trojan horse anche se il successo non è garantito). Selezionare questo significa che i giochi che utilizzano file esterni non verranno eseguiti correttamente. L'impostazione funziona solo mentre GameMaker è in esecuzione. Quindi se esegui il gioco indipendentemente da GameMaker, per esempio da un eseguibile stand-alone, non verrà eseguito in modalità sicura.
  • Show news on startup. Se mostrare le news di GameMaker una volta al giorno quando il programma viene eseguito. (Non può essere cambiato nella Lite Edition).
  • Show news in default brower. Se mostrare le news di GameMaker nel tuo browser internet di default, piuttosto che nel programma stesso. (Non può essere cambiato nella Lite Edition).
  • Location of GameMaker TV style Logo. Nella Lite Edition un piccolo logo verrà mostrato nello spigolo dell'area del gioco quando crei un eseguibile. Questo ti permette di scegliere lo spigolo.

Forms tab

Sprite Form

  • Show the origin in the sprite image. Se selezionato, nella finestra delle proprietà degli sprite, nell'immagine sprite, verrà indicata l'origine dello sprite.

Object form

  • In object properties, show hints for actions. Se selezionato, nella finestra delle proprietà degli objects, quando terrai il tuo mouse sopra una delle azioni, verrà mostrata una descrizione.
  • Default Tab to select. Questo ti permette di scegliere la linguetta default delle azioni quando viene aperta la finestra degli object. Questo è utile solo se usi sempre l'azione codice selezionando la linguetta 4, risparmiandoti di prenderla ogni volta. Questa opzione funziona nelle Timelines...

Room Form

  • When closing, remove instances outside the room. Se selezionato, il programma ti avvertirà quando ci sono delle istanze o tiles fuori da una room e ti permetterà di rimuoverle.
  • Remember room settings when closing the form. Se selezionato, alcune impostazioni delle room, come la visibilità della griglia, o la cancellazione degli oggetti sottostanti, verranno ricordate quando modificherai la stessa room in seguito.
  • Room Grid ON by default. Questo ti permette di disattivare la griglia della room ogni volta che una room viene aperta o creata.
  • Fill room with colour when no background colour selected.  Se crei una room e non le dai un colore di riempimento verrà riempita normalmente con un colore definibile dall'utente. Se deselezioni questa casella vedrai la griglia definibile dall'utente di default nella room. Questo può a volte essere utilizzato per visualizzare un livello.

Scripts and code tab

Questa impostazioni sono correlate all'editor di codice built-in. La maggior parte di queste impostazioni verranno spiegate in dettaglio nella pagina Scripts. Qui diamo giusto una piccola osservazione:

  • Group undo operations Quando delle azioni multiple (come scrivere un'intera linea) diventano una sola operazione per undo.
  • Number of undo Puoi cambiare questo numero, ma in generale questo dovrebbe andar bene.
  • Automatic indentation When set Enter and Backspace keys cleverly deal with indents.
  • Indent amount Quanto indentare con ogni tab.
  • Smart tabs When set tabs go to the position below the first non-empty space in the previous line.
  • Allow cursor beyond end of line When checked you can place the cursor at any position in the editor screen, also beyond the end of line. If unchecked this is not possible. This also changes the behavior of the left and right arrow keys.
  • Show auto-completion options When set help is given on which functions and variables start with the letters you are typing.
  • Dealy (msec) The delay in milliseconds before the help is shown. Set e.g. to 0 for immediate.
  • Show function argument help When set, when typing function arguments, in the statusbar help about this is provided.
  • Show find string When set, whenever typing in a find string, all locations are shown in the code.
  • Show line numbers Whether the editor should display the line numbers.
  • Show matching brackets Whether the editor should indicate matching open en close brackets.
  • Show code snippets with F2 When set, pressing F2 will bring up a number of useful code snippets.
  • Check code while typing When set the code is constantly checked and errors reported.
  • Enable BOLD on script keywords Allows you to have all script keywords in BOLD. Please select a FONT where the width of the BOLD string, is the same as the width of the NON-BOLD string.
  • Use color coding Whether to use color coding in the editor.
  • Colors Here you can set the colors used for color coding. You can also reset the default colors.
  • Font Here you can select the font for the editor. Note that only fixed pitch fonts are shown. Also note that only the font and the size are used.

Editors

Here you can choose whether to use the built-in code editor (highly recommended) or an external editor. In the case of an external editor, the script text is saved as a .txt file and the editor is called with this file as an argument. After editing it you should save the file and close the editor such that GameMaker can read the file back in.

Also you can choose whether to use the built-in image editor (highly recommended) or an external editor. In the case of an external editor, the image is saved as a .png file and the editor is called with this file as an argument. After editing it you should save the file and close the editor such that GameMaker can read the file back in.

Finally you can indicate the different external editors for the sound files you want to use. GameMaker does not contain internal sound editors. There are three possible editors handling .wav, .mid and .mp3 files. In the case of an external editor, the sound is saved as a .wav, .mid, or .mp3 file and the corresponding editor is called with this file as an argument. After editing it you should save the file and close the editor such that GameMaker can read the file back in.

Other Settings

It is also possible to change the location of the temp files that GameMaker creates to e.g. store the game that is tested. This location cannot be changed from the Preferences but is changes by creating a file tempdir.txt in the folder where GameMaker is located. In this file there should be one line containing the location of the folder in which the temp files are creates, for example C:\GMTEMP. Note that the name must include the complete path and should not end with a backslash. Each time GameMaker is run a unique temporary folder is created within this location. So things will work out fine, even when multiple instances of GameMaker use the same location. (Note that the games themselves also create a temp folder for their files. Again, this can be changed by creating a file tempdir.txt and placing it in the folder where the game executable is located.)

Secondly, it is possible to change the image GameMaker is using for the background of its main form. To this end place an image file back.bmp in the folder where GameMaker is located.
