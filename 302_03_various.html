Varie Altre Opzioni

Cancello subito questa riga, è solo che ho sbagliato il Commit Summary. Qui puoi impostare alcune opzioni addizzionali. Prima di tutto puoi impostare qualche tasto predefinito:

Let <Esc> end the game
Lascia che <Esc> termini il gioco Quando è spuntato, premere <Esc> causerà la chiusura del gioco. I giochi avanzati di solito non lo vogliono perché vorrebbero fare qualcosa (come salvare) prima di chiudere il gioco. In questo caso, leva il segno di spunta e dai le tue azioni per il tasto <Esc>.

Treat the close button as <Esc> key
Tratta il pulsante Chiudi come il tasto <Esc> Quando è spuntato il pulsante Chiudi sulla finestra genera un <Esc> Key Event. Quando NON è spuntato genera invece un Close Button Event (sotto Other Events).

Let <F1> show the game information
Lascia che <F1> mostri le informazioni del gioco Quando è spuntato F1 mostrerà le informazioni del gioco.

Let <F4> switch between screen modes
Lascia che <F4> scambi tra le modalità dello schermo Quando è spuntato F4 scambierà tra le modalità schermo intero e finestra.

Let <F5> save the game and <F6> load a game
Lascia che <F5> salvi il gioco e che <F6> lo carichi Quando è spuntato il giocatore può usare <F5> per salvare la situazione corrente e <F6> per caricare l'ultimo gioco salvato. (Nota che vengono salvate solo le informazioni di base. Quando usi funzionalità avanzate come gli effetti particellari o le strutture di dati queste impostazioni non vengono salvate e dovrai crearti da solo un meccanismo di salvataggio.)

Let <F9> take a screenshot of a game
Lascia che <F9> prenda un'istantanea del gioco Quando è spuntato il giocatore può premere <F9> per creare un'istantanea del gioco. Quest'istantanea viene salvata nel file screenshotXXX.png nella cartella da cuiil gioco sta venendo eseguito, dove XXX è il numero dell'istantanea. Se crei più istantanee il numero viene incrementato automaticamente.

Also you can set the priority of the game process. This priority indicates how much processor time is allotted to the game. In normal mode the operating system tries to give processor time to each process that needs it in some reasonable way. The higher you put the priority the more time is allotted to the game, making it run more smoothly and faster. But other processes get less time (also Windows processes so even the mouse might not move anymore). Use this with care.

Finally, you can set the version information of the game. This information is shown in the helptip that pops up when the user rests his mouse on the game executable. It is also shown when the users clicks with the right mouse button on the program and chooses properties and then version. If you distribute your game you best fill in this information.

You should give your game a version number that consists of four parts: the major version, the minor version, the release number and the build number. Also you should provide the name of the company (you), the name of the product, the copyright information, and a very brief description. All these fields should be at most 64 characters long.
